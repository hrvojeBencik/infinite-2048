## Spacing Scale

Declared values (multiples of 4, derived from existing codebase usage):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, card inner padding |
| md | 16px | Default element spacing, horizontal page padding |
| lg | 24px | Section padding, button horizontal padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: minimum 44dp height for all tappable controls (iOS HIG compliance for store submission)
- Button vertical padding: 14px (existing AppTheme `elevatedButtonTheme` — intentional, not an 8pt multiple)
- Card border radius: 16px (existing `cardTheme`) — keep unchanged in paywall
- Dialog border radius: 20px (existing `dialogTheme`) — keep unchanged

Source: `app_theme.dart` (ElevatedButton padding: horizontal 24, vertical 14; card radius 16; dialog radius 20).

## Color

Source: `app_colors.dart`. Do not introduce new hex values. Map all paywall elements to existing tokens.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0A0E21` (AppColors.background) | Paywall page background, screenshot backgrounds |
| Secondary (30%) | `#16213E` (AppColors.surface) | Paywall content card, pricing card background |
| Accent (10%) | `#6C63FF` (AppColors.primary) | See reserved list below |
| Premium accent | `#FFD700` (AppColors.secondary) | Paywall premium badge, star icons, trial CTA highlight |
| Destructive | `#FF5252` (AppColors.error) | Destructive actions only (none in paywall; restore purchase is neutral) |
| Text primary | `#FFFFFF` (AppColors.textPrimary) | All primary paywall copy |
| Text secondary | `#8D8D8D` (AppColors.textSecondary) | Cancel instructions, legal fine print |

Accent (`#6C63FF`) reserved for:
- Primary "Subscribe" / "Start Free Trial" CTA button background
- Active selection ring on plan option cards
- Hyperlinked text (Terms of Service, Privacy Policy links)

Premium accent (`#FFD700`) reserved for:
- "Premium" badge / crown icon on paywall header
- Trial period highlight text (e.g. "7-day free trial")
- Gradient on premium CTA when trial is available (`AppColors.premiumGradient`)

Adaptive icon background color (STORE-01, D-02 from CONTEXT.md):
- Use `#0A0E21` (AppColors.background) as the Android adaptive icon background layer
- This matches the app's dominant surface color and ensures a consistent brand identity when the icon appears on any launcher background

## Paywall Compliance UI Contract (STORE-10)

Apple Guideline 3.1.2 requires that the following be visible without scrolling on the paywall screen:

| Requirement | UI Element | Specification |
|-------------|-----------|---------------|
| Full annual price | Price display text | Display size (24sp, bold, `#FFFFFF`), positioned in top half of card |
| Trial terms | Trial copy block | Body size (16sp, regular), e.g. "7-day free trial, then $X.XX/year" |
| Cancel instructions | Cancel copy | Body size (16sp, regular, `#8D8D8D`), e.g. "Cancel anytime in App Store settings" |
| Terms of Service link | Tappable text | Label size (14sp, `#6C63FF`), minimum 44dp touch target height |
| Privacy Policy link | Tappable text | Label size (14sp, `#6C63FF`), minimum 44dp touch target height |

Layout constraint: All five elements above MUST be visible on first render without scrolling on the smallest required device (iPhone SE, 375pt wide, 667pt tall). The paywall content card must fit within the visible viewport at this size.

Scroll behavior: The paywall MAY be scrollable for expanded feature lists, but the pricing/trial/cancel block must not be behind a scroll boundary.

## Screenshot Visual Contract (STORE-02)

Screenshots are captured manually from running simulators/devices (D-04 from CONTEXT.md). The following visual state is required for each scene.

### Scene 1: Active Gameplay
- Board state: Active 4x4 grid with visible high-value tiles (512, 1024 minimum present)
- Score display: Non-zero score (target: 4000+ to appear impressive)
- Power-up bar: All four power-ups shown in their enabled (non-greyed) state
- Zone context: Genesis Zone (purple accent, `#6C63FF`) for brand color consistency
- Background gradient: `AppColors.backgroundGradient` visible

### Scene 2: Level Select / Zone Overview
- Zone cards: All 5 zones visible with their respective accent colors
- Progress indicators: At least 2 zones showing partial completion (stars visible)
- No dialogs or overlays — clean zone grid

### Scene 3: Level Complete Dialog
- Confetti burst active (or captured mid-animation)
- Star rating: 3 stars displayed
- Score and XP shown
- Share button visible

### Screenshot framing
- No device frames applied (submit raw screenshots to both stores — App Store Connect and Play Console apply their own device frames for featured artwork)
- No caption overlays or marketing text overlaid on screenshots (store guidelines vary; plain gameplay screenshots are safest for v1.2)
- Status bar: hidden or clean (dark, no carrier text, time 9:41 for iOS convention if simulator shows it)

## Copywriting Contract

This phase has no new screens except the paywall compliance fix. Copy below governs the paywall widget and any error/empty states introduced.

| Element | Copy |
|---------|------|
| Paywall primary CTA (trial available) | "Start Free Trial" |
| Paywall primary CTA (no trial) | "Unlock Premium" |
| Paywall trial terms line | "Try free for 7 days, then {price}/year" |
| Paywall cancel instruction | "Cancel anytime in your App Store settings" |
| Paywall restore link | "Restore Purchase" |
| Paywall legal links | "Terms of Service" and "Privacy Policy" (separate tappable links) |
| Paywall error (purchase failed) | "Purchase could not be completed. Check your payment method and try again." |
| Paywall error (restore failed) | "No active subscription found for this Apple ID." |
| App Store title | "2048: Merge Quest" |
| App Store subtitle (iOS) | "Puzzle Game with Levels & Zones" |
| Play Store short description | "Classic 2048 puzzle with levels, zones, and daily challenges" |
| Privacy policy live URL | "https://hrvojebencik.github.io/infinite-2048/privacy-policy.html" |

Sources: D-08, D-09, D-12 from CONTEXT.md. ASO body copy (keywords, full descriptions) generated separately by `/aso-expert` skill into `STORE_LISTING.md` — not specified here.

Tone: Casual, playful, action-forward. No legalese in CTA buttons. Legal copy (cancel, terms) is concise and plain-language.

Destructive actions in this phase: None. "Restore Purchase" is a neutral action, not destructive — use outlined button style (not error color), no confirmation dialog required.
