**Step 2: Add PrivacyInfo.xcprivacy to Xcode Runner target**

This is critical — without this step, Apple will reject with ITMS-91053 (per RESEARCH.md Pitfall 3). The file must be referenced in `project.pbxproj` so Xcode includes it in the app bundle.

Open `ios/Runner.xcodeproj/project.pbxproj` and make the following edits:

1. **Add PBXFileReference** — In the `/* Begin PBXFileReference section */` block, add:
   ```
   A1B2C3D4E5F60001 /* PrivacyInfo.xcprivacy */ = {isa = PBXFileReference; lastKnownFileType = text.xml; path = PrivacyInfo.xcprivacy; sourceTree = "<group>"; };
   ```
   (Generate a unique 24-char hex ID that does not conflict with existing entries.)

2. **Add PBXBuildFile** — In the `/* Begin PBXBuildFile section */` block, add:
   ```
   A1B2C3D4E5F60002 /* PrivacyInfo.xcprivacy in Resources */ = {isa = PBXBuildFile; fileRef = A1B2C3D4E5F60001 /* PrivacyInfo.xcprivacy */; };
   ```

3. **Add to Runner PBXGroup children** — Find the PBXGroup whose `path = Runner` and add the file reference ID to its `children` array:
   ```
   A1B2C3D4E5F60001 /* PrivacyInfo.xcprivacy */,
   ```

4. **Add to PBXResourcesBuildPhase** — Find the `/* Resources */` build phase for the Runner target and add the build file ID to its `files` array:
   ```
   A1B2C3D4E5F60002 /* PrivacyInfo.xcprivacy in Resources */,
   ```

**Alternative approach if project.pbxproj editing proves fragile:** Use the `xcodeproj` Ruby gem or instruct the user via a checkpoint to open Xcode, right-click the Runner group, select "Add Files to Runner...", select PrivacyInfo.xcprivacy, ensure "Copy items if needed" is unchecked and Runner target is checked. However, attempt the programmatic approach first.
