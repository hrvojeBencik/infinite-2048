## Screenshots Checklist

You need screenshots for each device type. Recommended 5-8 screenshots.

### Recommended Screenshots (in order)

1. **Home Screen** -- Show the main menu with Play button, daily challenge, and premium banner
2. **Gameplay (Story)** -- An active 4x4 game board mid-level with score display
3. **Special Tiles** -- A level featuring bomb and ice tiles in action
4. **Level Complete** -- The confetti celebration dialog with 3 stars
5. **Zone Selection** -- The zone map showing all 5 zones with progress
6. **Endless Mode** -- The endless 2048 board with high score display
7. **Achievements** -- The achievements page showing unlocked and locked achievements
8. **Daily Challenge** -- A daily challenge in progress with time/move constraints

### Required Sizes

| Platform | Device | Size (px) |
|---|---|---|
| Play Store | Phone | 1080 x 1920 or 1242 x 2208 |
| Play Store | 7" Tablet | 1200 x 1920 |
| Play Store | 10" Tablet | 1600 x 2560 |
| App Store | iPhone 6.7" (15 Pro Max) | 1290 x 2796 |
| App Store | iPhone 6.5" (11 Pro Max) | 1242 x 2688 |
| App Store | iPhone 5.5" (8 Plus) | 1242 x 2208 |
| App Store | iPad Pro 12.9" | 2048 x 2732 |

### Screenshot Tips

- Use a dark background matching the app theme (#0A0E21)
- Add short captions above or below each screenshot (e.g., "50 Levels Across 5 Zones")
- Use consistent fonts and colors from the app's theme
- Show the app in its best state (high scores, many stars earned)
- Consider using a screenshot framing tool like Previewed or AppMockUp

## Privacy Policy and Terms

| Document | Notes |
|---|---|
| **Privacy Policy URL** | Required for both stores. Must be hosted on a public URL. Configured via Firebase Remote Config in the app (`privacy_policy_url` key). |
| **Terms of Service URL** | Recommended. Configured via Firebase Remote Config (`terms_of_service_url` key). |

Both links are fetched from Firebase Remote Config, so you can update them without releasing a new app version.

**What to include in your Privacy Policy:**
- Data collected: Firebase Auth (optional email), device identifiers for ads (AdMob), purchase data (RevenueCat)
- Third-party SDKs: Firebase, Google AdMob, RevenueCat
- Data storage: All game data stored locally on device (Hive)
- Children's privacy: App is rated Everyone/4+, no data collected from children
- Contact information

**Free generators:** Termly.io, PrivacyPolicies.com, or app-privacy-policy-generator on GitHub

> Provide these in the "Notes for Review" field when submitting to Apple.

```
This is a puzzle game based on the 2048 mechanic. The app works fully without
an account -- Firebase authentication is optional and only used if the user
chooses to sign in from the profile page.

The app uses Google AdMob for advertisements, which are removed with a premium
subscription via RevenueCat/StoreKit.

Test account: Not required. The app functions fully without login.
```

If you are using sandbox testing for IAPs, also note:

For subscription testing, please use a sandbox Apple ID. The subscription
offerings are configured in App Store Connect under the "premium" subscription
group.
